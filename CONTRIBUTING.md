# Contributing to BetterKV

Thanks for your interest in contributing! BetterKV is moving fast and contributions are very welcome.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a new branch for your change

## What to Work On

Check the open issues for tasks. Good first issues are labeled `good first issue`.

Areas that always need help:
- Redis command implementations
- Benchmarks and performance improvements
- Documentation
- Bug reports and reproductions

## Submitting a PR

- Keep PRs focused — one thing at a time
- Add tests where applicable
- Describe what your PR does and why in the description
- Reference any related issues

## Reporting Bugs

Use the bug report issue template. Include reproduction steps and your environment.

## Note

BetterKV is not yet production ready. Breaking changes can and will happen.
